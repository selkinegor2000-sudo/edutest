# Материалы для приложений и выбор основных файлов кода

Ниже приведен готовый план приложений для диплома. Он сделан по логике, похожей на ваш пример на фото: в приложения выносятся именно ключевые файлы кода, а не весь проект подряд. Это выглядит убедительнее и лучше соответствует содержанию работы.

## Какой код стоит выносить в приложения

Не нужно выносить все подряд. Для диплома лучше показать те файлы, которые действительно отражают архитектурное ядро системы:

1. Схема базы данных.
2. Подключение к базе данных.
3. Серверную логику API и интеграцию AI.
4. Основной файл клиентского приложения с маршрутизацией.
5. Страницу создания теста.
6. Страницу прохождения теста.
7. Страницу результатов с AI-анализом.

Такой набор покрывает все главное: данные, архитектуру, бизнес-логику, пользовательский сценарий и интеллектуальную часть.

## Готовый текст для описания скриншотов в практической части

### Описание панели преподавателя

На рисунке 6 представлена панель преподавателя, содержащая сводные показатели по разработанным тестам, числу студентов, общему количеству прохождений и среднему результату. Кроме карточек с основными метриками интерфейс включает графические элементы аналитики, позволяющие оценить результаты по отдельным тестам и общую динамику активности обучающихся.

### Описание формы создания теста

На рисунке 7 показан интерфейс создания теста. Форма поддерживает пошаговое заполнение основной информации о тесте, добавление вопросов разных типов, настройку временных ограничений и параметров публикации. Реализованный интерфейс позволяет преподавателю формировать как стандартные тесты, так и задания с открытым ответом, предназначенные для интеллектуального анализа.

### Описание страницы прохождения теста

На рисунке 8 представлен интерфейс прохождения тестирования студентом. В верхней части страницы отображаются таймер, индикатор прогресса и количество уже заполненных ответов. Дополнительно реализована навигация между вопросами, что повышает удобство выполнения теста и позволяет пользователю контролировать собственное продвижение.

Для открытых ответов предусмотрен отдельный блок интеллектуального комментария, формируемого языковой моделью.

### Описание AI-анализа

На рисунке 10 приведен пример AI-анализа открытого ответа студента. В данном блоке система отображает ответ обучающегося, эталонный вариант и комментарий интеллектуального модуля, содержащий оценку полноты и корректности ответа. Такой формат усиливает образовательную ценность системы, поскольку студент получает не только числовой результат, но и содержательную обратную связь.

## Готовый текст для демонстрации AI-анализа в работе

В разработанном прототипе интеллектуальная проверка применяется к вопросам открытого типа. После завершения теста серверная часть формирует запрос к языковой модели, включая формулировку вопроса, эталонный ответ преподавателя и фактический ответ студента. На выходе система получает числовую оценку в диапазоне от 0 до 100 и текстовое пояснение. Далее этот результат преобразуется в количество начисленных баллов и отображается пользователю на странице итогового отчета.

Практическая ценность данного механизма заключается в том, что студенту предоставляется не только итоговая оценка, но и развернутый комментарий, объясняющий сильные стороны ответа и аспекты, требующие доработки. Для преподавателя это снижает нагрузку при первичной проверке открытых вопросов, а для обучающегося делает процедуру контроля более понятной и полезной в дидактическом отношении.

## Что не стоит выносить в приложения

1. Мелкие UI-компоненты из папки components/ui.
2. Автогенерируемые конфигурационные файлы вроде tsconfig.json и vite.config.ts.
3. Вспомогательные хуки, если они не играют ключевой роли в логике диплома.
4. Файлы зависимостей и package-lock.

## Почему такой выбор приложений выглядит убедительно

Он показывает полный путь работы системы:

1. Сначала описывается структура данных.
2. Затем демонстрируется подключение к базе.
3. После этого показывается серверная логика и AI-интеграция.
4. Затем идет клиентская маршрутизация.
5. В финале демонстрируются ключевые пользовательские сценарии.

Для дипломной работы это выглядит логично, содержательно и по-человечески, а не как случайный набор файлов проекта.
